**Break Tide** (Ninja Action Attack, base 2p/3d) gains dominate mid-combat via Combo: "If Rushing River or Flood of Force was the last attack this combat chain, Break Tide gains +3{p}, dominate, and [banish effect]." (CR 8.4.1 Combo.)

Ruling: Combo abilities are checked and applied as Break Tide becomes a chain link (a static ability that's true or false based on game state), so if Break Tide is played as the 2nd attack after Rushing River/Flood of Force, it already has dominate when it becomes attacking — defenders can then only add ONE card from hand (CR 8.3.4a).

The hard case is the reverse order: if a defender has ALREADY added 2+ cards from hand to the chain link BEFORE any state grants Break Tide dominate (not possible from Combo itself since Combo is checked at chain-link-creation, but relevant if a different effect grants dominate to an attack later, e.g. mid-defense), CR 8.3.4c is explicit: "If an attack is defended by two or more cards from hand and then the attack gains dominate, no cards are retroactively removed from defending." Dominate only gates the ADD-DEFEND event (CR 8.5.32a) going forward — it never undoes cards already added.
