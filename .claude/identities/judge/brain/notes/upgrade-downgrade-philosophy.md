Upgrades/downgrades let a judge tune a penalty to the situation (PPG §1.4). An upgrade issues the next-more-severe penalty, a downgrade the next-less.

Two upgrade mechanisms:
1. Per-infraction upgrades/downgrades written into each infraction (e.g. Tardiness <1 min → Warning, >10 min → Game Loss; Missed Trigger → Warning if self-created with advantage).
2. Repeat-count upgrades by frequency in a tournament DAY, resetting each day of a multi-day event:
   - Gameplay infractions: 3+ of the SAME type in a day → +1 severity (except Failure to Maintain Game State) (§2).
   - Tournament infractions: 2+ of the SAME type in a day → +1 severity (except Tardiness and Slow Play) (§3).

Caps: gameplay and tournament infraction upgrades never exceed Game Loss (§1.4). When a penalty is upgraded/downgraded TO an IP, it's an IP2 (§1.4).

Multiple related infractions at once: apply each procedure but issue only the single highest penalty — don't stack (§1.2), see [[handling-an-infraction-workflow]]. Beyond these codified moves, only the Head Judge may [[deviations]]. Severity scale: [[penalty-severity-ladder]]. — PPG (latest, 2026-06-10).
