**Mutiny on the Battalion Barque** (Pirate Action): "For each hero that controls more Gold than you, steal a Gold token they control. If you gain control of 1 or more Gold tokens this way, your next attack this turn gets +2{p}." Note the card's OWN text uses "steal" and then immediately checks "gain control" as the resulting condition — this is not a coincidence of wording. Steal (CR 8.5.54): "To steal a card, the player gains control of the object." CR 8.5.54a: "The player that controls the effect is considered to have stolen." Gain/control (CR 8.5.35): "To gain control of an object, that player is considered to control the object instead of its previous controller."

Ruling: because CR 8.5.54 defines Steal AS an instance of gaining control, any separate effect worded "whenever you gain control of a card/token" (see e.g. **Kyloria**'s "gain control of an item...") is triggered by a Steal effect too — Steal and Gain are not mutually exclusive named mechanics for the purpose of "gain control" triggers. The asymmetry runs the other way: a plain Gain-control effect does NOT make the player "considered to have stolen" (only 8.5.54a attaches that specific status), so effects keyed specifically off "whenever you steal a card" would NOT fire from a generic gain-control effect like Kyloria's.

See [[kw-steal]], [[kw-gain-control]].
