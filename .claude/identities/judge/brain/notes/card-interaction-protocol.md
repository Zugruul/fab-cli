PROTOCOL — answering "how do cards A and B interact?" (follow ALL steps; the lean-recall failure mode is stopping after steps 1-2 and missing set-specific rulings): (1) CARD TEXT: card-vault brain first (card-<slug>-<color>.md), fallback corpus/live per ROLE.md lookup order; note each card's SETS from the note's frontmatter and its keywords. (2) KEYWORDS: recall the [[kw-*]] note for EVERY keyword/type mechanic on both cards (ward, go again, instant, aura, ...) via [[keywords-index]]. (3) RELEASE NOTES (the step that gets skipped): recall release-notes-<set> for EACH set either card belongs to, PLUS [[release-notes-index]] for supersession and cross-set principles — set-specific rulings (holo counters, attack queue, ward leave-arena timing, Starfall feeds) live ONLY there; a card printed in multiple sets needs each set's note. (4) COMBAT/TIMING: recall [[combat-chain-steps]], [[combat-step-rulings]], trigger/priority/LKI notes when the interaction touches the chain. (5) VERIFY: check precision-sensitive claims against third_party/fab-rules/en-fab-cr.txt with section cites; refresh via `fab-cli rules update-docs` if VERSIONS.txt shows >24h. (6) OFFICIAL PER-CARD RULINGS: search both cards at https://cardvault.fabtcg.com/. (7) HONESTY: separate cited rulings from grammar-derived conclusions; anything CR-silent goes to #ask-a-judge ([[escalation-ask-a-judge]]) — never guessed.
