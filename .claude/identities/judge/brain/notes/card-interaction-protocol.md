PROTOCOL — answering "how do cards A and B interact?" (follow ALL steps). STEP 0 — CACHE CHECK: look for an existing ix-<card-a>-x-<card-b> note (slugs alphabetical; ls notes/ | grep '^ix-'). If found AND its recorded CR build matches third_party/fab-rules/VERSIONS.txt, answer FROM IT; if the CR bumped, re-verify (steps 2-6) and RE-MINT with the new CR stamp. Then: (1) CARD TEXT: card-vault brain first (card-<slug>-<color>.md), fallback corpus/live per ROLE.md; note each card's SETS frontmatter and keywords. (1b) TRUE TEXT DOUBLE-CHECK (MANDATORY for precision-sensitive rulings): `fab-cli fabtcg card "<name>"` — official Card Vault TRUE text (the authority per CR 2.0.2) + printed-text diff + errata flags; the offline corpus and vault brain notes may lag behind errata. If true text differs from the brain's card note, TRUST TRUE TEXT and update the card note. (2) KEYWORDS: recall [[kw-*]] for EVERY keyword on both cards via [[keywords-index]]. (3) RELEASE NOTES (the step that gets skipped): recall release-notes-<set> for EACH set either card belongs to PLUS [[release-notes-index]] (supersession + cross-set principles); the release-notes frontmatter now lists cards:/keywords: covered — grep them: `grep -l "<card-slug>" notes/release-notes-*.md`. (4) COMBAT/TIMING: [[combat-chain-steps]], [[combat-step-rulings]], trigger/priority/LKI notes when the chain is involved. (5) VERIFY precision-sensitive claims against third_party/fab-rules/en-fab-cr.txt with section cites; refresh docs if >24h. (6) OFFICIAL RULINGS: `fab-cli fabtcg card "<name>" --json` → rulings_errata[], or cardvault.fabtcg.com. (7) HONESTY: separate cited from grammar-derived conclusions; CR-silent points marked UNSETTLED → #ask-a-judge ([[escalation-ask-a-judge]]). STEP 8 — STORE: mint/re-mint the ix- note: CR build stamp in source, UNSETTLED points inline, links to involved [[kw-*]] and [[release-notes-<set>]] notes. INVALIDATION: on new CR build or set ingest, grep ix-* notes for changed keywords and re-verify hits. Example: [[ix-fleeing-starbreeze-x-gone-in-a-flash]].
