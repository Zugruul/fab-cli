A triggered effect ("When/Whenever/At/The [ordinal] time…") creates a triggered-layer when its condition is met (CR 6.6.1, 6.6.5). It never uses "instead" — that's replacement, see [[replacement-effect-ordering]].

Adding to the stack (CR 6.6.6): the triggered-layer is added before the next player gets priority, as a game state process (see [[game-state-actions-and-death]]). The controller must declare modes and targets when it's added; with no legal target it ceases to exist and is not added (6.6.6a).

Ordering multiple simultaneous triggers (6.6.6b, mirrored in 1.10.2d): the turn-player picks a starting player, then each player in clockwise order stacks ALL their own triggered-layers in an order they choose. So a player orders their own triggers; the turn-player only decides who goes first.

Trigger nuances for rulings:
- Event triggers don't fire if the event is modified away before it occurs (6.6.5b).
- A trigger prevented from triggering still counts against its once-per-turn limit (6.6.5f).
- If a trigger is missed by inaction, that's a PPG infraction, not a rules event — see [[missed-trigger-timing-windows]]. — CR (latest, 2026-06-10).
