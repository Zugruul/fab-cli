Replacement application order per event (CR 6.5, deepens [[replacement-effect-ordering]]): turn-player selects a starting player once per event (6.5.2b); then in clockwise passes each player applies their own effects by TYPE: self-/identity-replacements → standard-replacements → prevention effects → THE EVENT OCCURS → outcome-replacements → complete (6.5.3-6.5.8). Core constraints: an effect can replace a given original event only once and never its own output (6.4.5); replacements never apply retroactively (6.4.3); sub-events (Ward destroying its source) occur before the main event and are themselves replaceable (6.4.1a). Self-replacement conditions evaluate when the preceding effect generates, or when its parameters are determined (Weave Earth fused-check, 6.4.7b). Prevention nuances for calls: prevention amounts recompute per application (Arcanite Fortress X, 6.4.2a); "prevents 1 less" applies after establishing total prevention (Vambrace, 6.4.10g); unpreventable damage still consumes fixed-preventions and their riders but not shield pools (6.4.10h). Modified events can newly trigger/activate other effects (6.4.6). Links: [[continuous-effect-staging-detail]], [[triggered-effects-and-ordering]].
