A replacement effect replaces an event with a modified event before it occurs (CR 6.4.1); "instead" and prevention/"prevent" are the textual markers (6.4.9, 6.4.10). It must exist before the event (6.4.3) and can only replace a given event once (6.4.5).

When multiple could apply to one event, order is fixed (CR 6.5): the turn-player selects a starting player, then in clockwise order each player applies all their effects of the current type before moving on (6.5.1–6.5.2). The type sequence is: self-/identity-replacement → standard-replacement → prevention → THE EVENT OCCURS → outcome-replacement → event complete (6.5.3–6.5.8).

Prevention specifics judges get asked about:
- Prevention amount defaults to the event's damage if unspecified (6.4.10b); unspecified type prevents all types (6.4.10c).
- Fixed-prevention is one-shot; shielding-prevention ("prevent the next X") persists across events until X is spent (6.4.10i–j).
- If damage "can't be prevented," the prevention still applies once but its amount isn't reduced and side-modifications still happen (6.4.10h).

Contrast with the continuous staging system in [[layers-and-continuous-effect-staging]] and triggers in [[triggered-effects-and-ordering]]. — CR (latest, 2026-06-10).
