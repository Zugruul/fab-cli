Named-event boundaries decide most trigger disputes (CR 8.5): "destroyed" ONLY via destroy effect; graveyard arrival by other means isn't destruction (8.5.4a) — same pattern for discard (8.5.5a), draw (8.5.6a), banish (8.5.1a), charge (8.5.29b), reveal (8.5.17a). Replacement keeps the name: redirected discard/banish/draw still count as that named-event (8.5.1b, 8.5.5c, 8.5.6c — pairs with CR 1.9.3a). Damage taxonomy: generic vs {p} vs arcane; cross-type triggers/preventions don't apply (8.5.3e-f); life loss ≠ damage ≠ hit (8.5.12b). Search failure legality: may fail only when properties are specified AND no public match (8.5.19a-b); property-less search on non-empty zone can't fail (8.5.19c). Copy/become: copyable = printed properties (+ creation properties for tokens); parameters copied; later changes to the reference don't propagate (8.5.25c); copying the same reference twice fails (8.5.25d). Failed-effect catalog for calls: draw/opt on empty deck fails; turn to same visibility fails; tap tapped fails; unfreeze unfrozen fails; look at public fails; equip needs empty zone(s) + right subtype (8.5.41b-c); steal/give/gain-control of equipment must re-equip or the whole effect fails (8.5.54b). Clash: no {p} property = other card wins; double-whiff = no winner (8.5.45b-c). Ignore vs reroll ordering: reroll first (8.5.33c).
