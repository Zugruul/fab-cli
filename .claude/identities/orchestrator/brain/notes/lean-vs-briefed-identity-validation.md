To validate a brain-backed identity, ask the SAME question twice via separate agents: once fully briefed (card texts, coverage checklist, pointers to the right notes) and once LEAN (bare question only, no hints). The briefed run establishes the knowledge ceiling; the lean run shows unaided behavior; the DELTA is a precise map of missing links or missing protocol steps — each miss is either a structural link to add ([[recall-needs-structural-links]]) or a protocol note to mint. Also validates honesty: a good identity separates cited claims from derived ones and escalates CR-silent points instead of guessing. Cheap, repeatable after every fix.
