When a design doc's Key-sequences step tells the dev agent exactly what the red-first structural/citation test should assert (required headings in order, citation-density thresholds, per-section claims) before any doc prose is written, 'TDD' on a pure-documentation task stops being theater and becomes a real red-to-green cycle with a falsifiable bar. Write that specificity into the design doc, not just into the dev-agent brief.
