A FAB card's printed properties (CR 2.0, 1.12) include: name; the colored pitch strip with socketed resource pips (pitch value — [[pitch-values-and-color]]); a resource cost (top-right, CR 1.14); power {p} (attack value) and defense {d} where applicable; a type line of types and subtypes (e.g. "Ninja Action - Attack"); supertypes = class + talents (e.g. Ninja, Shadow); rarity; and a functional text box with abilities and keywords. Not every card has every stat — equipment has {d} but no cost/pitch, resource cards only pitch. Numeric properties can never go below zero (CR 2.0.3c). Reading these correctly is the basis of evaluation and pitch planning. Read actual card text from the DB, never memory ([[card-corpus-and-search]]). Keywords live in the text box ([[keyword-go-again]] and siblings); classes/talents in the type line ([[classes-and-talents]]).
