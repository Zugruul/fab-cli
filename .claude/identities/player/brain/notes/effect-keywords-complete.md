Effect keywords define named-events precisely (CR 8.5) — the name matters for triggers ([[events-and-multi-events]]). Zone-movers: banish, destroy (→grave via destroy only), discard (from HAND only; redirected discard still "discarded", 8.5.5c), draw (top of deck→hand; empty deck = failed draw, NOT a loss — FAB has no draw-out death, 8.5.6b), put/return, charge (hand→soul), pitch (as an effect), transform (put under a permanent), transcend, exchange (simultaneous swap of zone/visibility/control). Damage system (8.5.3): three types — GENERIC "damage", PHYSICAL {p} (combat damage step), ARCANE (only from effects that say arcane); type-specific triggers/preventions NEVER cross types (8.5.3e-f); life LOSS is not damage and damage prevention can't stop it (8.5.12b). Info effects: look (no visibility change), reveal (public then private; position unchanged; reveal-until-condition that whiffs still triggers reveal-effects, 8.5.17e), opt N (top N: top/bottom in any order), name (full name, never moniker, must exist in the game, 8.5.21a-b). Searches: may fail ONLY if the spec names properties and no PUBLIC match exists (8.5.19a-c). Combat/control: attack-with, add-defend, negate (stack only), freeze/unfreeze, gain-control/steal/give (equipped things must re-equip or the effect fails), intimidate (random face-down banish, back at end phase; "intimidated" even with empty hand, 8.5.10a). Dice: roll, reroll (original never happened), ignore (post-hoc "never happened"; reroll before ignore, 8.5.33c). Misc: clash (compare top-card {p}; no power property = auto-lose, both whiff = nobody wins), wager, amp N (+N to next arcane damage event), mark, sharpen, crank, tap/untap, cheer/boo (they stack, don't cancel).
