When a layer resolves (CR 5.3.1): check-resolution → statics functional → resolution effects in printed order → GO AGAIN grants its action point LAST among effects → leave stack → clear. Attacks never "resolve" on the stack — they move to the queue for combat instead (5.3.1a). Failure checks that matter: a state-triggered layer (e.g. phantasm) RE-CHECKS its condition at resolution — pump the defender's {p} in response and phantasm fizzles (5.3.2a); a defense reaction that can no longer defend the chain link fails (5.3.2b). One effect failing doesn't stop the rest (5.3.4b). Go again uses LKI if the layer vanished (5.3.5a). Defense reaction card-layers become defending cards on resolution (5.3.6b).
